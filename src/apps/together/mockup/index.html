  • What is the name of your app?

    BombSquad.

  • What collaborative activity can people do together using your app?

    People will be able to play a game together by working together to defuse a fake bomb.

  • What is the ideal number of concurrent users

    The game should be able handle up to 40 players. However, in it's inital versions the ideal number will be 10-20 concurrent users.

  • What skills do people need in order to do this activity?

    The user should be able to quickly read text and be able to use audio. Also, users should be in a similiar space for optimal bomb defusing tactics.

  • Is this activity for fun or for something practical?

    This activity is just for fun.

  • How can a person see what everyone else is doing?

    Initially, the person will see a completion bar that fills as each user completes their task. A goal for the app is to have a real time 'picture' or 3D shape, that has buttons, switches and lights. As users complete their tasks, these toggles will flip to the 'correct' state, indicating to current users how close the bomb is to being defused.

  • How can a person see the most recent result of everyone’s collaborative effort?

    During a game, each person will see a 'real-time' picture of the current progress of the bomb defusal process. After completion, the game state could be saved with details about that specific bomb defuse.

  • What can an admin see (i.e., God’s mode)?

    The admin can see a current games list as well as details about any current games. They will be able to see the number of users in a game, the current status of the bomb defuse (i.e. which player has completed their part of the defusal process and who is still working), as well as the correct answers to finish the defusal of the bomb.
